H264概述

MPEG-4 AVC/H.264标准提高编码效率的主要改进技术包括：

  1. 采用可变图块大小的帧间预测和移动补偿，预测图块的大小不在局限于16*16像素，可小到4*4像素。
  2. 采用空间的帧内预测，预测图块的大小可以是16*16的宏块，也可以是4*4的图块，定义了多种预测方式，目的是找到相关性最大的预测。
  3. 采用DCT演进的“整数变换编码”，提高运算速度
  4. 熵编码采用效率更高的CAVLC（context-based adaptive variable length coding）以及CABAC(context-based adaptive binary arithmetic coding)。

三种类型的图像：

  1. I帧 (intra-picture):包含完整图像，用于其他帧的解码参考
  2. P帧（predicted-picture):以在它之前出现的帧内图像I作参考图像，对P帧的编码即对他们之间的差值多编码
  3. B帧（bidirectionally-interpolated picture)：以他之前的I帧以及之后的P帧图像做参考

MPEG AVC/H.264编解码的过程包括：帧内预测、帧间预测、变换、量化、熵编码

MPEG AVC/H.264标准分成两个层次：

  1. VCL（Video Coding Layer）:用于有效表达视像内容
  2. NAL（Network Abstration Layer)：按照一定格式组织VCL数据并提供header等信息，便于在不同的网络上进行传输

其中NAL在网络传说中，封装成NALU的形式，即header+data。header的格式通常为0x00 00 00 01 type
